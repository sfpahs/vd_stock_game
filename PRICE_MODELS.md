# 자산 타입별 가격 결정 공식

## 개요
주식, 코인, 환율 각 자산 타입이 시간에 따라 어떻게 가격이 결정/변동되는지를 정의합니다.

---

## 1. 주식(STOCK) 가격 변동 모델 [KRW 기준]

### 모델: 기하 브라운 운동 (Geometric Brownian Motion, GBM)

**수학 공식**:
```
P_stock(t) = P_0 × exp(μ_s × t + σ_s × √t × Z_s)

여기서:
- P_stock(t): 시간 t의 주식 가격 (KRW)
- P_0: 초기 가격 (예: 삼성전자 78,000 KRW)
- μ_s: 드리프트(추세) = ln(1 + 0.0005) ≈ 0.0005 (일일 0.05% 상승)
- σ_s: 변동성(volatility) = 0.02 (일일 2% 변동성)
- Z_s: 표준 정규분포 난수 N(0,1)
- t: 경과 시간 (시간 단위)
```

**코드 구현용 단순화**:
```cpp
// 매 1시간 갱신
double drift = 0.0005;          // 0.05% 상승 추세
double volatility = 0.02;       // 2% 변동성
double randomNoise = gaussRandom();  // N(0,1)
double changeRate = drift + volatility * randomNoise;
double newPrice = currentPrice * (1 + changeRate);
```

**특징**:
- ✅ 평균적으로 소폭 상승 (장기 우상향)
- ✅ 일일 변동폭: ±2%
- ✅ 극단적 변동 제한: P(t) ≥ P_0 × 0.5 (최대 50% 하락선)
- ✅ 균형잡힌 시뮬레이션 게임성

## 2. 코인(COIN) 가격 변동 모델 [USD 기준]

### 모델: 고변동성 기하 브라운 운동

**수학 공식**:
```
P_coin(t) = P_0 × exp(μ_c × t + σ_c × √t × Z_c)

여기서:
- P_coin(t): 시간 t의 코인 가격 (USD)
- P_0: 초기 가격 (예: BTC 45,000 USD)
- μ_c: 드리프트 = ln(1 + 0.001) ≈ 0.001 (일일 0.1% 상승)
- σ_c: 변동성 = 0.08 (일일 8% 변동성) ← 주식의 4배
- Z_c: 표준 정규분포 난수 N(0,1)
```

**코드 구현용 단순화**:
```cpp
// 매 1시간 갱신
double drift = 0.001;           // 0.1% 상승 추세 (약함)
double volatility = 0.08;       // 8% 변동성 (높음)
double randomNoise = gaussRandom();  // N(0,1)
double changeRate = drift + volatility * randomNoise;
double newPrice = currentPrice * (1 + changeRate);
```

**특징**:
- ✅ 주식 대비 4배 높은 변동성 (±8%)
- ✅ 추세는 약함 (거의 무작위 변동)
- ✅ 극단적 변동 가능: 하루에 ±30% 급등/급락
- ✅ 고수익/고위험 시뮬레이션

## 3. 환율(FX) 변동 모델 [KRW/USD]

### 모델: 평균 회귀형 기하 브라운 운동

**수학 공식**:
```
E(t) = E_long + (E_0 - E_long) × exp(-λ × t) + σ_e × √t × Z_e

여기서:
- E(t): 시간 t의 환율 (KRW/USD)
- E_long: 장기 평균 환율 = 1,200 KRW/USD (기준값)
- E_0: 초기 환율 = 1,200 KRW/USD
- λ: 회귀 속도 = 0.0001 (느린 회귀)
- σ_e: 변동성 = 0.005 (일일 0.5% 변동성) ← 주식의 1/4
- Z_e: 표준 정규분포 난수 N(0,1)
```

**코드 구현용 단순화**:
```cpp
// 매 1시간 갱신
double meanReversion = 0.0001;  // 회귀 속도
double volatility = 0.005;      // 0.5% 변동성
double randomNoise = gaussRandom();  // N(0,1)

// 평균으로 천천히 회귀
double revertRate = -meanReversion * (currentRate - 1200.0);

// 변동
double randomChange = volatility * currentRate * randomNoise;

double newRate = currentRate + revertRate + randomChange;

// 범위 제한: 너무 극단적으로 가지 않도록
newRate = clamp(newRate, 1100.0, 1300.0);  // ±100원 범위
```

**특징**:
- ✅ 주식/코인 대비 극히 낮은 변동성 (±0.5%)
- ✅ 장기 평균(1,200원)으로 회귀하는 특성
- ✅ 극단적 편차 자동 조정 → 현실적
- ✅ 실제 FX 시장 특성 반영

## 4. 변수 정리

| 자산 타입 | 초기가 | 드리프트(μ) | 변동성(σ) | 특징 |
|----------|-------|-----------|---------|------|
| **주식(STOCK)** | P_0 | +0.0005 | 0.02 | 안정적, 우상향 |
| **코인(COIN)** | P_0 | +0.001 | 0.08 | 고변동성, 무작위 |
| **환율(FX)** | 1,200 | -0.0001* | 0.005 | 저변동성, 평균회귀 |

*환율은 평균회귀 모델 사용

```

   - 모든 코인 가격 갱신
   - 환율 갱신

2. 매 종목마다:
   - 난수 생성: Z = N(0,1)
   - 변동률 계산: Δ = μ + σ × Z
   - 새가격 = 기존가격 × (1 + Δ)
   - 필요시 범위 제한

3. UI 갱신:
   - 포트폴리오 테이블 새로고침
   - 현재가 라벨 갱신
   - 손익 재계산
```

### 난수 생성 (가우시안)
```cpp
// C++에서 표준정규분포 난수 생성
#include <random>

std::random_device rd;
std::mt19937 gen(rd());
std::normal_distribution<> dis(0, 1);  // N(0,1)

double Z = dis(gen);  // 표준 정규분포 난수
```

## 6. 이중통화 영향

### USD 자산의 환율 민감도
```
코인_평가액(KRW) = 코인_가격(USD) × 수량 × 환율(KRW/USD)

예시:
- 코인 가격: 45,000 USD
- 수량: 0.5
- 환율: 1,200 KRW/USD
→ 평가액 = 45,000 × 0.5 × 1,200 = 27,000,000 KRW

환율이 1,200 → 1,250으로 상승 (USD 강세):
→ 새 평가액 = 45,000 × 0.5 × 1,250 = 28,125,000 KRW
→ 환율 손익 = +1,125,000 KRW (코인 가격 미변)
```

### 포트폴리오 손익 분해
```
총_손익 = 
  + 주식_가격_손익 (주식 가격 변동)
  + 코인_가격_손익(USD) (코인 가격 변동, USD 기준)
  + 코인_환율_손익 (환율 변동, USD 자산에만 영향)
```

## 8. 참고 사항

### 가우시안 난수의 특성
```
Z ~ N(0,1)이면:
- P(-1 < Z < 1) ≈ 68% (1σ)
- P(-2 < Z < 2) ≈ 95% (2σ)
- P(-3 < Z < 3) ≈ 99.7% (3σ)

따라서 변동률 = μ + σ × Z:
- 68% 확률로 [μ-σ, μ+σ] 범위
- 95% 확률로 [μ-2σ, μ+2σ] 범위
- 극단치(5σ) 거의 불가능
```

### 실제 시장과의 차이점
```
✅ 실제 반영:
  - GBM 모델 사용 (주식/코인 시장 표준)
  - 평균회귀 환율 모델
  - 로그 정규분포 가격 변동

❌ 단순화:
  - 일괄 변동 (실시간이 아님)
  - 고정 파라미터 (시간대별 변동성 미적용)
  - 거래량/스프레드 미반영
```
